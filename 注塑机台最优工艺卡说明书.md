## 1. 系统概述

### 1.1 系统定位

**注塑机台最优工艺卡分析系统**是一个基于 Streamlit 的 Web 应用程序，专为注塑车间的生产工艺优化而设计。系统的核心目标是：**从海量的历史生产数据中，自动识别出每个机台稳定高效运行的时段，并提取该时段内的工艺参数作为"最优工艺卡"的推荐值**。

### 1.2 系统能解决什么问题

| 问题 | 传统做法 | 本系统的做法 |
|------|---------|-------------|
| 如何确定最优工艺参数？ | 依赖老师傅经验，凭感觉调整 | 基于大量数据自动计算，用统计方法给出推荐值 |
| 某台机器什么时候运行最好？ | 无从追溯，只能看当班记录 | 滑动窗口扫描全部历史数据，精确定位最优时段 |
| 哪些参数对 OEE 影响最大？ | 靠猜测，没有量化依据 | Pearson 相关系数定量分析每个参数与 OEE 的关系 |
| 各机台的稼动率模式是怎样的？ | 接口头汇报，缺乏可视化 | 热力图、柱状图、饼图多维度直观展示 |
| 如何生成多机台的最优工艺卡？ | 逐台手动整理 Excel | 一键扫描全部机台，自动生成推荐值 |

### 1.3 核心工作流程

```
上传两个报表 → 系统清洗+关联 → 浏览工艺趋势 → 分析稼动率 → 设定条件 → 自动生成最优工艺卡
```

## 2. 核心概念与术语

### 2.1 OEE（设备稼动率）

在注塑行业上下文中，**设备稼动率**（OEE, Overall Equipment Effectiveness）是衡量设备实际运行效率的核心指标。

$$OEE = \frac{有效生产时长}{总开机时长}$$

- 理论上 OEE 取值范围为 [0, 1]（即 0% ~ 100%）
- OEE 越高，表示设备越充分地用于生产
- OEE 低可能由多种原因导致：离线（故障维修）、待机（无工单/换模）、报警（异常停机）等

### 2.2 "最优工艺卡"的定义

本系统中的"最优工艺卡"是指：

> **在 OEE 持续 ≥ 设定阈值（默认 90%）的最长连续生产时段内，各工艺参数的中位数作为推荐值，Q25 ~ Q75 作为建议操作范围。**

这个定义隐含了两个核心逻辑：

1. **高 OEE 时段 = 生产稳定时段**：OEE 持续高说明设备在该时段内几乎没有异常停机，是一次"顺畅的生产"
2. **稳定生产的参数 = 最优参数**：在顺畅生产时段使用的工艺参数，就是使生产过程最优的参数组合

### 2.3 工艺参数分组

系统将注塑机的所有工艺参数分为四大类：

| 参数组 | 包含参数 | 典型数量 |
|--------|---------|----------|
| **射胶参数** | 射胶压力一段~五段、射胶速度一段~五段、射胶位置一段~五段 | 15个 |
| **保压参数** | 保压压力一段~三段、保压速度一段~三段、保压时间一段~三段 | 9个 |
| **温度参数** | 温度一段值~六段值 | 6个 |
| **其他参数** | 储料背压、射退位置、射退压力、射退速度、储料时间、射胶终点、成型周期 | 7个 |

合计共 **37 个工艺参数**。

### 2.4 关键阈值

| 阈值名称 | 默认值 | 含义 |
|---------|--------|------|
| OEE 达标阈值 | 0.90 (90%) | OEE 必须 ≥ 此值才视为"稳定生产" |
| 最小连续时数 | 12 小时 | OEE 达标必须连续超过此时长才被选为"最优窗口" |
| 数据断点检测 | 1.5 小时 | 连续两行时间差 > 1.5 小时视为数据缺口，窗口断开 |
| OEE 超高异常 | 1.5 (150%) | OEE > 150% 视为数采异常 |
| OEE 负值 | < 0 | 负值视为数采异常 |

## 4. 数据导入模块详解

### 4.1 报表格式要求

系统需要用户上传两类 Excel 报表文件：

#### 4.1.1 工艺趋势报表

此报表记录各机台在每个小时段的工艺参数实测值。

| 必需列 | 数据类型 | 说明 |
|--------|---------|------|
| `日期` | 文本/日期 | 数据的日期，系统自动标准化为 `YYYY-MM-DD` |
| `机台号` | 文本 | 唯一标识每台注塑机 |
| `时间段` | 文本 | 格式如 `14:00-15:00`，表示该小时段 |
| 各类工艺参数列 | 数值 | 射胶参数、保压参数、温度参数、其他参数，共计约37列 |

此外可能包含：`车间`、`产线`、`固资编码`、`机型`、`最后数采时间` 等信息列。

#### 4.1.2 稼动率小时报表

此报表记录各机台在每个小时段的设备运行效率数据。

| 必需列 | 数据类型 | 说明 |
|--------|---------|------|
| `日期` | 文本/日期 | 数据的日期 |
| `机台号` | 文本 | 唯一标识每台注塑机 |
| `时间段` | 文本 | 格式如 `14:00-15:00` |
| `设备稼动率` | 数值 | 该小时的设备综合效率 OEE |
| `有效生产时长(min)` | 数值 | 实际生产分钟数 |
| `离线时长(min)` | 数值 | 因故障/维修离线的分钟数 |
| `待机时长(min)` | 数值 | 无工单/换模/等待的分钟数 |
| `报警时长(min)` | 数值 | 异常报警停机分钟数 |
| `开机时长(min)` | 数值 | 设备通电开机总分钟数 |
| `开合模次数` | 数值 | 该小时开合模总次数 |

### 4.2 数据加载流程

当用户同时上传两个 Excel 文件后，系统执行以下处理流程：

```
第1步 ──→ 读取工艺趋势报表 Excel 文件
         │  ├── 调用 load_process_report() 函数
         │  ├── 使用 pandas.read_excel() 解析
         │  ├── normalize_date_str() 标准化日期列 → "YYYY-MM-DD"
         │  ├── normalize_time_slot() 标准化时间段列
         │  └── 机台号转为字符串并去空格
         │
第2步 ──→ 读取稼动率小时报表 Excel 文件
         │  ├── 调用 load_utilization_report() 函数
         │  ├── 同样标准化日期、时间段、机台号
         │  └── 强制转换数值列为数字类型
         │
第3步 ──→ 两表关联 merge_reports()
         │  ├── 关联键: ['机台号', '日期', '时间段']
         │  ├── 使用 INNER JOIN（只保留两表都有匹配的行）
         │  └── 生成 _timestamp 列（日期 + 时间段起始小时 → datetime）
         │
第4步 ──→ 异常标记 flag_abnormal_oee()
         │  ├── OEE < 0  → 标记为"负数异常"
         │  └── OEE > 1.5 → 标记为"超高异常"
         │
第5步 ──→ 存入 session_state 供各页面共享
         │
第6步 ──→ 保存本地磁盘缓存（.cache/目录）
```

**日期标准化**：系统通过 `normalize_date_str()` 函数将各种日期格式（如 `2024-01-15`、`2024/1/15`、Excel 日期序列值等）统一转换为 `YYYY-MM-DD` 字符串。

**时间戳生成**：关联完成后，系统从 `日期 + 时间段起始小时` 生成 `_timestamp` 列，例如 `2024-01-15` + `14:00-15:00` → `datetime(2024, 1, 15, 14, 0)`。

### 4.3 数据清洗与异常标记

系统通过 `flag_abnormal_oee()` 函数自动识别并标记两种 OEE 异常：

| 异常类型 | 判定条件 | 含义 | 新增标记列 |
|---------|---------|------|-----------|
| **负数异常** | `设备稼动率 < 0` | 传感器/数采系统异常 | `oee_abnormal = True`<br/>`oee_abnormal_type = '负数异常'` |
| **超高异常** | `设备稼动率 > 1.5` | OEE 理论上 ≤ 1.0，> 1.5 明显是采集故障 | `oee_abnormal = True`<br/>`oee_abnormal_type = '超高异常'` |
| **正常** | `0 ≤ 设备稼动率 ≤ 1.5` | 正常数据，允许轻微超出 1.0 | `oee_abnormal = False`<br/>`oee_abnormal_type = '正常'` |

> **注意**：异常数据不会从分析中被移除，而是被标记。在后续分析中会进行裁剪（clip）处理，将 OEE 值限制在 [0, 1] 区间，防止异常值影响统计结果。

### 4.4 两表关联逻辑

关联是通过 **`merge_reports()`** 函数实现的：

1. **定义关联键**：`['机台号', '日期', '时间段']` 三个字段
2. **标准化验证**：确保三个关键列在两表中都是去空格的字符串
3. **列去重处理**：稼动率报表中删去与工艺报表重复的列，避免合并时出现 `_x` / `_y` 后缀
4. **INNER JOIN**：只保留两表同时存在的记录
5. **失败诊断**：如果关联后为空，系统自动展示两表的关键列样本值，帮助定位差异

关联成功的三个条件：
- **机台号** 完全一致（包括字母大小写和前后空格）
- **日期** 标准化后能对应
- **时间段** 格式一致（如都是 `14:00-15:00`）

### 4.5 本地缓存机制

系统设计了一套完整的本地持久化缓存机制：

**缓存保存**：首次加载数据后自动将 `merged_df` 以 pickle 格式存入 `.cache/merged_data.pkl`，同时在 `.cache/meta.txt` 中记录缓存时间、源文件名、行数和机台数。

**缓存恢复**：每次启动时自动检测缓存文件，若存在则一键恢复到 `session_state`，无需重新上传。

**缓存清除**：点击"清除数据"按钮后，删除磁盘缓存并清空 session_state。

**分析级缓存**：在最优工艺卡分析中，还使用基于数据指纹的缓存键，相同数据+相同条件直接返回缓存结果，避免重复计算。

### 4.6 数据质量报告

加载完成后自动生成质量报告：

| 指标 | 说明 |
|------|------|
| 关联后数据行数 | 两表关联成功的数据总行数 |
| 可分析机台数 | 两表中同时存在数据的机台数量 |
| 覆盖天数 | 数据的日期跨度 |
| OEE 异常记录数 | 被标记为异常（负数或超高）的记录条数 |

## 5. 工艺趋势分析模块详解

该模块提供三种分析模式：单机台趋势、多机台对比、参数-OEE 相关性。

### 5.1 单机台趋势分析

#### 5.1.1 控制面板

- **选择机台**：从下拉列表中选择要分析的注塑机台号
- **参数类别**：通过复选框选择要查看的参数组（射胶/保压/温度/其他），可多选
- **日期范围**：通过滑块选择分析的时间窗口

#### 5.1.2 工艺参数趋势图

使用 Plotly 生成交互式折线图：

- **双Y轴设计**：温度参数使用右轴展示，其他参数使用左轴，解决量纲差异问题
- **曲线区分**：非温度参数用实线+圆形标记点；温度参数用虚线+三角标记点
- **交互功能**：鼠标悬停显示精确值、框选缩放、图例切换、一键导出 PNG 图片

#### 5.1.3 参数分布箱线图

通过箱线图（Box Plot）展示各参数的分布特征：Q25、中位数、Q75、须线、均值±标准差标记。

#### 5.1.4 机台统计信息

| 统计项 | 含义 |
|--------|------|
| 总数据时数 | 该机台在数据中的总记录条数 |
| 平均稼动率 | 所有记录中 OEE（已裁剪到 [0,1]）的均值 |
| OEE ≥ 90% 时数 | 稼动率达到 90% 以上的时段数 / 总时数 |
| OEE 异常值数 | 被标记为异常的记录数 |

#### 5.1.5 参数数值摘要表

以表格形式展示每个参数的平均值、中位数、标准差、最小值、最大值、样本数。支持一键下载为 CSV。

### 5.2 多机台对比分析

用于对比 2~6 台机台在同一工艺参数上的趋势差异：

- **对比折线图**：不同机台使用不同颜色，X 轴时间 / Y 轴参数值，统一悬停模式
- **对比统计表**：每个机台的参数平均值、中位数、标准差、平均 OEE、样本数

### 5.3 参数-OEE 相关性分析

这是本模块最有分析价值的功能——**量化哪些工艺参数对 OEE 的影响最大**。

#### 5.3.1 计算方法

使用 **Pearson 相关系数** 衡量每个工艺参数与设备稼动率之间的线性关系：

$$r = \frac{\sum_{i=1}^{n}(X_i - \bar{X})(Y_i - \bar{Y})}{\sqrt{\sum_{i=1}^{n}(X_i - \bar{X})^2}\sqrt{\sum_{i=1}^{n}(Y_i - \bar{Y})^2}}$$

其中 $X$ = 工艺参数值，$Y$ = 设备稼动率。$r$ 取值范围为 [-1, 1]。

#### 5.3.2 相关系数解读

| r 值范围 | 相关性强度 | 含义 |
|---------|-----------|------|
| \|r\| > 0.7 | 强相关 | 该参数对 OEE 有显著影响 |
| 0.4 < \|r\| ≤ 0.7 | 中等相关 | 该参数对 OEE 有一定影响 |
| 0.2 < \|r\| ≤ 0.4 | 弱相关 | 该参数对 OEE 影响较弱 |
| \|r\| ≤ 0.2 | 极弱/无关 | 该参数与 OEE 基本无关 |

#### 5.3.3 相关性方向

- **正相关（r > 0，绿色）**：参数值越大，OEE 越高
- **负相关（r < 0，红色）**：参数值越大，OEE 越低
  - 如：储料时间过长 → 节拍变慢 → OEE 降低

#### 5.3.4 相关性柱状图

按相关系数绝对值降序排列的横向条形图。绿色=正相关，红色=负相关，标注精确的 r 值。

#### 5.3.5 数据质量要求

参与相关性计算必须满足：
- 参数至少有 5 个有效值（dropna后）
- OEE 与该参数均有值的共同数据点 ≥ 5 个
- 相关系数计算后需 pd.notna() 验证

## 7. 最优工艺卡生成模块详解（核心）

这是整个系统最核心的功能模块。

### 7.1 筛选条件设置

在分析前，用户需设定四个关键参数：

#### 7.1.1 最少连续生产时数（默认 12 小时）

定义了一段"稳定生产"必须持续的最小长度。

**为什么需要？** 偶尔一两小时 OEE 高可能是巧合，只有长时间持续高 OEE 才说明该工艺参数组合确实有效。

**选择建议**：
- 数据充足：推荐 12~24 小时
- 数据有限：可降到 6~8 小时
- 短周期产品（薄壁件）：可适当降低
- 长周期产品（厚壁件）：建议保持较高值

#### 7.1.2 OEE 阈值（默认 90%）

每小时稼动率 ≥ 此值才视为"达标"。

**选择建议**：
- 90%（默认）：严格优良标准
- 85%：适中标准
- 80%：宽松标准

#### 7.1.3 选择机台范围

- **全选**：一次性分析所有机台
- **手动选择**：指定特定机台分析

#### 7.1.4 开始分析按钮

点击后系统执行完整扫描流程。

### 7.2 滑动窗口扫描算法

由 `find_optimal_windows()` 函数实现，核心流程：

```
输入: 机台 X 的全部数据（已按时间排序）

    i = 0
    while i < n:
        if 第i小时达标:
            j = i
            while j < n and 第j小时达标:
                if j > i:
                    time_gap = (第j行时间 - 第j-1行时间) / 3600 小时
                    if time_gap > 1.5:  ← 数据缺口，窗口断裂
                        break
                j += 1
            
            if (j - i) >= min_hours:
                记录窗口 [i:j] → 存入结果列表
            
            i = j  ← 从窗口末尾继续扫描
        else:
            i += 1

输出: 按 (持续时长↓, 平均OEE↓) 降序排列的窗口列表
```

**关键设计——数据缺口检测**：

```
数据行:  08:00, 09:00, 10:00, [缺失11:00采集], 12:00, 13:00
                                 ↑ time_gap = 2小时 > 1.5
窗口只到 10:00，12:00 开始是新窗口
```

这保证了"最优窗口"内数据是真正连续的，而非中间跳过了一大段缺失数据。

**OEE 裁剪处理**：达标判定前，OEE 值先裁剪到 [0, 1] 区间：
```python
df['_oee_clipped'] = df['设备稼动率'].clip(0, 1)
df['_qualified'] = df['_oee_clipped'] >= oee_threshold
```

### 7.3 最优窗口选取策略

按**两级优先级**排序：

```
第一优先级: 持续时长（duration_hours）最长  ← 最看重
第二优先级: 平均 OEE（avg_oee）最高        ← 同时长时的选择
```

即 **"先找最长稳的，再从中选最稳的"**。

```python
windows.sort(key=lambda w: (w['duration_hours'], w['avg_oee']), reverse=True)
best_window = windows[0]  # 排序后第一个即为最优
```

### 7.4 工艺参数提取方法

找到最优窗口后，`compute_optimal_params()` 函数从窗口数据中提取参数统计值：

#### 推荐值 = 中位数（Median）

**为什么是中位数？**
- 不受极端值影响（鲁棒性好）
- 即使窗口内有少量异常读数，推荐值仍然稳定
- 代表了"最典型"的参数设置

#### 建议范围 = [Q25, Q75]（四分位距 IQR）

表示在稳定生产期间，该参数 **50% 的时间** 都落在这个区间内。

#### 完整输出统计指标

| 统计量 | 含义 | 用途 |
|--------|------|------|
| **推荐值** ⭐ | 中位数 | 最优工艺卡核心推荐值 |
| **下限(Q25)** | 第25百分位 | 建议操作范围下限 |
| **上限(Q75)** | 第75百分位 | 建议操作范围上限 |
| **平均值** | 窗口内均值 | 与中位对比——偏差大说明分布偏斜 |
| **标准差** | 波动程度 | 标准差小=参数控制稳定；大=有波动 |
| **最小值** | 窗口内最小值 | 确认参数从未低于此值 |
| **最大值** | 窗口内最大值 | 确认参数从未超过此值 |
| **有效样本数** | 参与统计的数据点数 | 确认样本量是否充足 |

#### 参数排序规则

最终工艺卡按预定义的参数分组顺序排列（射胶→保压→温度→其他），而非按字母顺序。

## 10. 常见问题与最佳实践

### 10.1 数据关联失败

**排查清单**：
1. 两个文件都有 `日期`、`机台号`、`时间段` 列
2. 机台号格式一致（`A01` vs `A 01` vs `A-01`）
3. 日期格式标准化后可对应
4. 时间段格式一致

关联失败时系统会展开调试面板，展示两表关键列样本值。

### 10.2 没有找到达标窗口

| 原因 | 表现 | 解决方案 |
|------|------|----------|
| OEE 阈值太高 | 全车间 OEE 普遍 85% | 降到 0.85 或 0.80 |
| 连续时数太大 | 有达标但不够 12h | 降到 6~8h |
| 数据缺口多 | 热力图中大量空白 | 检查数采系统连续性 |
| 确实无稳定生产 | 热力图大多红黄色 | 管理层面改善 |

### 10.3 推荐值 vs 平均值

| | 推荐值（中位数） | 平均值 |
|---|---|---|
| 受极端值影响 | ❌ 不受影响 | ✅ 受影响 |
| 代表什么 | "最典型"的参数值 | "平均"的参数值 |
| 为什么用 | 更鲁棒，更安全 | 作为参考对比 |

### 10.4 相关性分析的局限性

- Pearson 系数只衡量**线性关系**，U 型关系（如温度太低太高都不好）可能漏掉
- 相关 ≠ 因果，高相关性不等于调它能改善 OEE
- 参数变化范围很小时，相关系数不可靠

**建议**：将相关性分析作为"发现候选参数"的辅助工具，配合工艺知识和现场实验验证。

### 10.5 最佳实践总结

1. **数据量要充足**：至少覆盖 3 天以上数据
2. **先全后分**：先全选所有机台扫描，再针对具体机台深挖
3. **多轮迭代**：先用默认参数，太多机台无结果再逐步放宽
4. **结合现场经验**：推荐值是数据驱动的，最终确认前请师傅审核
5. **定期更新**：积累新数据后重新分析
6. **优先关注强相关参数**：在调机时优先调整与 OEE 强相关的参数
7. **横向对比**：同型号机台推荐值差异大 → 排查是否存在特殊问题
